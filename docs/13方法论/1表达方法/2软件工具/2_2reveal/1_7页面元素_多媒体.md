



# [多媒体Media](https://revealjs.com/media/)



## [自动播放](https://revealjs.com/media/#autoplay)

1. 局部自动播放: `<video data-autoplay></video>`
2. 全局自动播放: `Reveal.initialize()`传入` autoPlayMedia: null`参数, 该参数控制video/audio/iframe的自动播放. null: 只在data-autoplay时自动播放. true: 所有的都自动播放, 无论自身的设置. false: 所有的都不自动播放, 无论自身的设置.

## [全屏显示图片视频和连接Lightbox](https://revealjs.com/lightbox/)

在需要全屏显示的元素中添加如下的属性:

1. data‑preview‑image
2. data‑preview‑video
3. data‑preview‑link ✨可以在`Reveal.initialize()`传入`previewLinks`.

```html
<img src="reveal.png" data-preview-video="video.mp4">
<a href="https://hakim.se" data-preview-link>Open Link</a>
```

## [懒加载Lazy Loading](https://revealjs.com/media/#lazy-loading)

reveal只有在距离当前最近的几页才加载图片, 音频, 视频和iframe. 在`Reveal.initialize()`中传入`viewDistance: 3`来控制距离的长短. 移动设备上通过`mobileViewDistance`来控制.

开启懒加载的方法是将图片, 音频, 视频和iframe的`src`属性改为`data-src`属性.

```html
<section>
  <img data-src="image.png">
  <iframe data-src="https://hakim.se"></iframe>
  <video>
    <source data-src="video.mp4" type="video/mp4" />
  </video>
</section>
```

✨iframe的懒加载问题:

一些iframe可能包含音频和视频文件, 在iframe那一页还未到达当前页时, 就会听到声音. 可以通过`data-preload`属性来重写🔴.另外可以在`Reveal.initialize()`中传入`preloadIframes`参数来全局控制, 该参数包含如下参数:

1. null: 在data-src和data-preload属性时, 在viewDistance加载. 只有data-src时在visible时加载. 
2. true: 所有带data-src的在viewDistance都会被加载. 
3. false: 所有带data-src只有在visible时被加载.
